# Timescale Data Processor

## Описание
Проект выполнен в рамках тестового задания на стажировку. Система предназначена для импорта, обработки и анализа timescale-данных. Реализована загрузка .csv файлов (с возможностью масштабирования и поддержкой других форматов) с валидацией данных, расчёт интегральных метрик и клиент-серверное взаимодействие через REST API. Также есть возможность развёртывания всего приложения в Docker.

## Технологии
- **Backend**: `.NET 10`, `ASP.NET Core Web API`, `EF Core`
- **Database**: `PostgreSQL`
- **Frontend**: `Angular 21`, `Nginx`
- **Контейнеризация**: `Docker`, `Docker Compose`
- **Testing**: `xUnit`, `Moq`
- **API Documentation**: `Swagger`

## Запуск
1. Склонируйте репозиторий и перейдите в корневую директорию:
   ```bash
   git clone https://github.com/jxhnsilver/timescale-data-processor.git
   ```
3. **Docker Desktop** (Windows/macOS) или **Docker Engine & Docker Compose** (Linux) должен бысть установлен и запущен.
4. Скопируйте файл конфигурации окружения с тестовыми данными:
   ```bash
   cp .env.example .env
   ```
5. Соберите образы и запустите все сервисы из корневой директории:
   ```bash
   docker compose up --build
   ```
6. Откройте интерфейсы:
   - Client (SPA): http://localhost:80
   - API (Swagger): http://localhost:8080/swagger
  
## Реализовано

- **Импорт CSV**: Парсинг файлов с разделителем `;` и валидация (диапазоны дат, типы значений, лимит строк). Транзакционная запись в БД с автоматическим откатом при ошибках. Поддержка перезаписи существующих файлов. Архитектура парсера позволяет легко добавлять новые форматы файлов.
- **Расчет метрик**: Автоматическое вычисление интегральных результатов при импорте: дельта времени, среднее/медианное значение показателя, минимальные и максимальные значения, время первой операции.
- **REST API**: Три основных метода — импорт файла (`POST /api/values/import`), фильтрация результатов (`GET /api/results`) и получение последних 10 записей по файлу (`GET /api/values/latest/{fileName}`).
- **SPA интерфейс**: Веб-интерфейс на Angular для загрузки файлов, просмотра и фильтрации результатов. Корректная обработка и отображение ошибок сервера.
- **Docker-развертывание**: Полная контейнеризация backend, frontend и базы данных. Многоэтапная сборка образов, проксирование API через `Nginx`, автоматические миграции БД и healthcheck'и для контроля готовности сервисов.
- **Unit-тесты**: Покрытие бизнес-логики валидации и расчета метрик с использованием `xUnit` и `Moq`.
